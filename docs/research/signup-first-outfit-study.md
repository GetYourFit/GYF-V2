# Signup-to-First-Outfit Study

Run 5-10 observed sessions before building analytics infrastructure.

Ask permission to observe/record actions; never record passwords or OTPs. Give one uncoached task:

> Create an account, tell GYF enough to style you, find one outfit you would consider wearing,
> and save it.

Moderator stays silent. Help only after participant is blocked for 60 seconds; count each assist.
Record timestamps, repeated clicks, visible errors, route, device, viewport, network, signup and
onboarding completion, and participant-confirmed useful outfit. Stop/delete recording on request.

Ask afterward: most useful outfit and why; least useful and why; expected behavior that did not
happen; biggest hesitation; what would make GYF useful weekly.

Use `signup-first-outfit-study.csv`. Report counts and median times, not inflated percentages from a
small sample. Do not put lifecycle events into `/feedback`; those are ML labels. Build separate
product analytics only after observed sessions confirm funnel definitions.
